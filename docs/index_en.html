DOSBox‑X ASM 2D Draw App

Intel 8086 (16‑bit) • DOSBox‑X • TASM

1. General Description

Modular x86 assembly application providing an interactive menu for drawing 2D shapes in graphics mode (13h). The project demonstrates the use of keyboard input, display (text and graphics), and memory (data segment, VRAM A000h).

  • Microprocessor: Intel 8086 (real mode)
  • Peripherals: keyboard (INT 21h/01h, INT 16h), display (INT 10h), memory (A000h)
  • Execution Environment: DOSBox‑X

2. Peripherals and Services

  • Keyboard: read character (DOS INT 21h/01h), wait for key (BIOS INT 16h/00h)
  • Display: text mode 80×25 and graphics mode 13h 320×200×256 (BIOS INT 10h)
  • Memory: data segment DS, video memory ES=A000h for pixels

Interrupts used:

  • INT 10h – AH=00h (set video mode), AH=02h (cursor position), AH=06h (scroll/clear)
  • INT 21h – AH=09h (print string), AH=01h (read char), AH=4Ch (exit)
  • INT 16h – AH=00h (wait for key)

3. Block Diagram

Modular structure: MAIN (initialization) → MENU/DRAWM (UI) → DRAWL/DRAWS (drawing algorithms) supported by IO (DOS/BIOS services), INP (numeric input), DATA (messages/constants).

4. Operation Diagram

  1. Program start: initialize DS, clear screen, display main menu.
  2. Navigation: user selects an option (Line/Square/Back/Exit).
  3. Parameter input: numeric input in text mode.
  4. Graphics mode: switch to 13h, draw the shape in VRAM.
  5. Return: wait for a key press, return to text mode.

5. Test Plan

Recommended scenarios for functional validation:

  1. Horizontal line: y=50, x=10, width=100, color=12 → drawn on row 50.
  2. Vertical line: x=100, y=20, height=80, color=10 → drawn on column 100.
  3. Filled square: (x,y)=(40,40), size=40, color=4 → area correctly filled.
  4. Square outline: (x,y)=(60,30), size=50, color=14 → only borders drawn.
  5. Invalid input: non-digit characters or out-of-range → re-prompt or ignore.
  6. Mode transitions: after drawing, key press → returns to text mode, screen cleared.

6. Implementation Details

  • Mode 13h: 320×200, 256 colors; video segment ES=A000h.
  • Pixel: offset = y × 320 + x; write color byte.
  • Interfaces: INT 10h (video), INT 21h (I/O), INT 16h (BIOS keyboard).
  • Modules: see header comments in ASM files.

7. Build & Run

Useful commands using TASM:

tasm /m2 application\\MAIN.ASM
tasm /m2 application\\MENU.ASM
tasm /m2 application\\DRAWM.ASM
tasm /m2 application\\DRAWL.ASM
tasm /m2 application\\DRAWS.ASM
tasm /m2 application\\INP.ASM
tasm /m2 application\\DATA.ASM
tasm /m2 application\\IO.ASM
tlink /t MAIN MENU DRAWM DRAWL DRAWS INP DATA IO

Run: use DOSBox‑X and launch the resulting executable.
